Command: channel


Syntax: channel subcommand ?options?

See the documentation for archive on how to obtain a channelId.
See also: value.

Details

  • Syntax: channel valid channelId
    Returns: 0 or 1
    Use this in loops over channels to check if the channelId references a valid entry.
    The following subcommands are only defined when a channelId is valid! 
    

  • 
Syntax: channel name channelId

    
Returns: string

    

  • 
Syntax: channel next channelId

    
Returns: 0 or 1

    
Make channelId reference next channel
(in no specific order).

    
The return values is the same as channel valid

    

  • 
Syntax: channel getFirstTime channelId

    
Returns: timestamp

    
Get stamp for when this channel was first archived.

    

  • 
Syntax: channel getLastTime channelId

    
Returns: timestamp

    
Get timestamp for when this channel was last archived.

    

  • 
Syntax: channel getFirstValue channelId

    
Returns: valueId

    
Return first value that's in the archive.

    

  • 
Syntax: channel getLastValue channelId

    
Returns: valueId

    
Return last value that's in the archive.

    

  • 
Syntax: channel getValueAfterTime channelId timestamp

    
Returns: valueId

    
Return first value in the archive that's following the given timestamp.

    

  • 
Syntax: channel getValueNearTime channelId timestamp

    
Returns: valueId

    
Return first value in the archive that's following the given timestamp.

    

  • 
Syntax: channel getValueBeforeTime channelId timestamp

    
Returns: valueId

    
Return last value in the archive that's immediately before the given timestamp.

    

  • 
Syntax: channel close channelId

    
Invalidate channelId when no longer used.
    
(Does not return error when Id was invalid)
