Command: value


Syntax: value subcommand ?options?


See also: archive, channel.

Details

  • Syntax: value valid valueId
    Returns: 0 or 1
    Use this in loops over values to check if the valueId references a valid entry.
    The following subcommands are only defined when a valueId is valid! 
    

  • 
Syntax: value isInfo valueId

    
Returns: 0 or 1

    
Some values are not numeric values but purely informational.

    
In that case value status  returns

    

      

    • Archive_Off

    • Disconnected

    • Disabled

    • ... maybe more

    

    
To check for such "info" values, this command can be used.

    

  • 
Syntax: value get valueId

    
Returns: numeric/double, list if valueId references array

    
Get numeric value of current valueId.

    
(arrays/lists not implemented, yet)

    

  • 
Syntax: value text valueId

    
Returns: text

    
Get current value of current valueId formatted as text.

    
For enumerated values this function will return
the state text assiciated with the value
while value get would only provide
the integer code.

    
(arrays/lists not implemented, yet)

    

  • 
Syntax: value time valueId

    
Returns: time-string as "yyyy/mm/dd hh:mm:ss.nanosec"

    
Get time stamp of current valueId.

    
The time string is in an ASCII-sortable format with 24h hours,
which is useful for display mostly.
For further calculations you might have to split it into numbers.

    

  • 
Syntax: value status valueId

    
Returns: status-string

    
Get status of current valueId.

    

  • 
Syntax: value next valueId

    
Returns: 0 or 1

    
Make valueId reference next value

    
The return values is the same as value valid

    

  • 
Syntax: value prev valueId

    
Returns: 0 or 1

    
Make valueId reference previous value

    
The return values is the same as value valid

    

  • 
Syntax: value type valueId

    
Returns: string describing type

    
Get type of current value for valueId.

    

  • 
Syntax: value close valueId

    
Invalidate valueId when no longer used.
    
(Does not return error when Id was invalid)
