Archive Engine

This new engine is intended to replace "chan_arch", the solaris-only prototype implementation of the ChannelArchiver data sampling engine.
It reads the same configuration files as the current engine, it should also generate binary-compatible data files.

Improvements over chan_arch:

  • Portable: Running on Win32, Linux, Solaris.
  • Build-in X11 GUI replaced by HTTP server
    • no longer limited to single status GUI per engine
    • portable
    • networked
    • use any web-browser for a status display by pointing it to
      http://<machine where engine is running>:<engine's port>
      The port defaults to 4812 unless changed by a command line option.

Implementation Details

I/O Library

The Engine uses the ChannelArchiver I/O library which handles portable access to the data storage.

Configuration Interface

While the engine does by default read the original configuration files, the new system also updates those files in order to make interactive changes persistent. It is also prepared for replacing this file-based configuration system with e.g. an approach that keeps configuration information in a database (See API).
