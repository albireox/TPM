ATAC: A TCL Archive client

Why TCL?  -  Usage -  Example Scripts

Motivation

Common requests to the archive are:

  • "How often was value x above 10 in the last week?"
  • "Show me x times y whenever z is on!"
  • ...

Another possible application is the interactive or automated generation of dayly reports.
Images of those plots can then be put on Web pages etc.:

Requests like these will never be easy to answer with a standard GUI tool, instead they require some sort of programming or query language.

Why TCL?

Short answer: Because it was of immediate use at LANL.

More reasons: Tool languages like Perl or TCL lack the computing power of SDDS, MatLab, ....,
but most programmers know them and with such a scripting language at hand you can generate other formats as needed.

TCL was one of the first embeddable scripting langages, it now runs (together with tk for a GUI) on Unix and Win32. So do Perl and Python. Now also with a GUI, in fact the very same tk, though that's faily new and less well known and supported. While I really like Perl, it's emphasis is still on text processing, TCL is faster for numerics.

A common problem with tcl was that extensions had to be compiled in, some were mutually exclusive or you could not easily use one extension without getting many others as well. Since version 8.0 dynamic loading is supported.

So TCL was chosen. Perl, Python, ... might still follow, but not at a high priority

Usage

After staring tcl (or wish, tcldp, bltwish, ...), load the atac extension: 

    source atacTools.tcl


More details on the supported commands can be found here.
